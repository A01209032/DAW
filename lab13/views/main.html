¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Session_unset() limpia el arreglo $_SESSION y session_destroy() destruye el archivo de la sesion pero no limpia las variables, por eso es importante borrar primero los datos de la sesion antes de temrinarlo porque asi no se queda basura en el sistema

¿Cuál es la diferencia entre una variable de sesión y una cookie?

Los datos en la cookie se guardan indefinidamente en la computadora del usuario mientras que la variable de sesion se borra cuando se cierra una sesion

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Se le asigna un nombre unico a todas las fotos para que no ocurran errores

¿Qué es CSRF y cómo puede prevenirse?

Cross-Site Request Forgery

Es un ataque que sua una sesion de un usuario para cambiar datos de la aplicacion, para prevenirlo se recomienda "token based mitigation"
